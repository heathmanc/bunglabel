BungVision Label Studio v0.9.33

Launcher/module path fix:
- Removed reliance on the user's current working directory for locating the bundled bung_labeler package.
- main.py now forces the extracted application folder onto sys.path before importing the app.
- bung_labeler/ui/main_window.py now also inserts the project root so direct troubleshooting launches do not fail with "No module named bung_labeler".
- Updated Windows and Linux launch scripts to set PYTHONPATH to the extracted folder and run main.py by absolute path.
- Added RUN_BUNGVISION_LABEL_STUDIO.bat and BungVision_Label_Studio.py as straightforward launch options.

Preserved:
- Strict reviewed-only export and training rebuild behavior.
- OBB battery + bung workflow.
- Basler/Pylon fixes and exposure controls.
- UI polish from v0.9.27+.
