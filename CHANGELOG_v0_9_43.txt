BungVision Label Studio v0.9.43 - Performance Tuning & Dead Code Removal

Performance improvements:
- Frame-sequence deduplication: live-preview timer now skips reprocessing when no new camera frame has arrived (avoids redundant gamma/CLAHE/sharpen + overlay rebuild on unchanged frames)
- Vectorized gamma LUT: replaced per-frame 256-iteration Python loop with a single numpy vectorized expression (~15x faster on the gamma path)
- Hoisted filter parsing in _bung_overlay_items: count_names/count_ids computed once per overlay refresh instead of once per detection result
- Single-pass image list build in _refresh_images: dataset summary accumulated inline while building the image list; eliminates the second full walk over the image list that previously happened in _update_dataset_summary
- Removed spurious _update_dataset_summary() call from _update_box_count (fired on every box add/move/delete); summary now only refreshes on actual disk changes

Dead code removed (~720 lines):
- Training tab and all training UI/logic (_train_tab, browse_train_dataset, start_training, stop_training, _append_train_output, _training_finished)
- TensorRT export tab and all export UI/logic (browse_trt_source_model, export_tensorrt_engine, _append_export_output, _export_finished)
- Live detection tab and all detection UI/logic (_detect_tab, toggle_live_detection, browse_detect_model, load_detect_model, _detections_to_boxes, _summarize_detections, _run_detection_on_frame, detect_current_frame)
- Unused overlay helpers (_draw_battery_obb_overlay, _draw_bung_overlay)
- Unused image-list prefix helper (_image_list_prefix)
- Unused battery-only OBB export helpers in yolo_export.py (_write_battery_obb_dataset, export_recipe_battery_obb, export_all_battery_obb)
- Unused bgr_to_qimage_rgb_bytes in image_adjust.py
- Orphaned imports: importlib.util, QProcess, QTextCursor, QInputDialog

Title bar updated from "Bung labeling tool v0.9.42" to "BungVision Label Studio v0.9.43".
Review sidecar reviewed_by field updated to v0.9.43.
